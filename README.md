# Sample of how deterministic guardrails fit with Human-in-the-loop safety controls

SAMPLE of how deterministic guardrails fit with Human-in-the-loop safety controls. This is NOT a complete workflow! DO NOT USE!



* To ensure that deterministic guardrails can overcome an LMM's native tendency to agree with the user, they must be placed before the LMM in the workflow.
* In order to meet new regulations, the LMM must have a real-time log of reasoning that is anchored to external truth.'
* The anchor to external truth is necessary. Without it, the LMM will default to its internal judgment, which is always 100%.
* For high-risk data, this real-time log must be monitored by 2 natural humans who can stop the work if the LMM begins to drift. 
* This requires a stop button that is available on the log page and is capable of stopping the LMM.
* All outputs for high-risk data also require a 2-human review process.
